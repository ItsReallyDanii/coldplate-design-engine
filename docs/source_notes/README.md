# Source Notes

File naming and ID rules:
- Name files as `LIT-001_short-topic.md`, `LIT-002_short-topic.md`, etc.
- Source IDs are assigned sequentially and are never reused.
- Retired or dropped sources keep their original ID.
- One note file corresponds to one source.

Minimal workflow:
1. Assign the next Source ID.
2. Add a row to `docs/reading_queue.md`.
3. Create a source note in `docs/source_notes/` from `docs/source_capture_template.md`.
4. Capture metrics and details in the note.
5. Update `docs/literature_matrix.md`.
6. Optionally link to a claim-audit entry later if the source becomes claim-relevant.
